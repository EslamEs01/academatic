# Contract: Static HTML-First & Django-Ready (Spec 008)

**Status**: Binding · The architecture contract for the Reports shell. Every category card / overview tile / per-area section / signal chip / action / planned-state is **baked static HTML** at build; runtime JS enhances only. References FR-014 / FR-015 / FR-016; SC-005; data-model §1–§13.

---

## SD0. Scope

Governs the regenerated `public/reports.html` (+ `.en`). It inherits the Spec 001–007 SSG rules verbatim. **No new public page; no `build-html.mjs`/`nav.config.js`/`dashboard.js` change.**

## SD1. Static HTML-first

No whole-page `<div id="app">` mount. `reports.html` is a **complete pre-rendered** static document: full shell (sidebar + topbar) + the entire reports body baked — page header, action cluster, operations overview, every category card, every per-area section, the planned/backendRequired cards, the filter bar. View Source shows it all without JS.

## SD2. Category cards + overview + sections are baked

`pageHeader` + `reportActions()` + a `summaryCards`-style operations overview + a `cardGrid` of `reportCard`s + the per-area summary sections + the planned cards + a `filterBar` — all built at **build time**. `enhance.js` only filters the pre-rendered cards, opens action confirms/disabled-reasons, and shows toasts — it **constructs no card/section/tile/chip DOM**.

## SD3. Reuse the closed `data-*` hook allowlist — NO new hook

Spec 008 introduces **no new `data-*` hook**. It reuses only:

```text
data-filter-form / data-filter / data-filter-reset / data-filter-apply / data-filter-count / data-no-results   (filter the category cards)
data-row + facet attrs (data-area / data-availability / data-signal / data-search)                              (filterable cards)
data-demo-action / data-toast                                                                                   (Print = demo)
data-confirm (+ -title/-msg/-cta/-toast)                                                                        (Schedule = confirm → demo)
data-disabled-reason / data-reason-key                                                                          (Export CSV/PDF / Share = disabled-with-reason)
```

Real `<a href>` (the source-page drill-downs + the breadcrumb) are navigation, not hooks. Planned/backendRequired cards reuse the existing `report-card` **disabled** variant (a non-`<a>` `aria-disabled` block with a `title` reason) — never a dead `<a href="#">`.

## SD4. Per-language pages

`reports.html` (ar/rtl) + `reports.en.html` (en/ltr), regenerated by the existing `build-html.mjs` `reports` entry (unchanged). Direction, digits, labels resolve per language. No runtime language fetch.

## SD5. i18n overlay (`ar.rep.js` / `en.rep.js`)

Reports strings live in a NEW overlay pair under the `rep.*` namespace, registered in `i18n.js` exactly like the existing `.crs`/`.trn` overlays:

```js
import arR from '../locales/ar.rep.js';
import enR from '../locales/en.rep.js';
// …
deepMerge(ar, arR);
deepMerge(en, enR);
```

`deepMerge` extends (never clobbers); a missing key renders `⟦key⟧` (smoke catches it). The legacy `reportsPage.*`/`report.*` keys that survive (page title/subtitle) stay; the removed finance `report.revenue.*` keys are dropped.

## SD6. Relative paths / no CDN / no library

`./assets/` relative paths only; `.nojekyll`; zero external/CDN requests; **no chart/table/form/calendar/analytics library** loads. GitHub-Pages compatible; opens directly from the filesystem.

## SD7. Build pipeline

`renderReports` (rewritten) is already registered in `build-html.mjs`; `npm run build` regenerates `reports.html` + `.en` and prints the page count (unchanged total — no new page); the nav guard passes (no nav change).

## SD8. Django-template readiness

| Static shape | Django |
|---|---|
| `reports.html` | `templates/admin/reports.html` |
| category cards | `{% for category in report_categories %}` → `_report_card.html` |
| operations overview | `{% for tile in academic_ops %}` |
| per-area sections | `{% for section in report_sections %}` → `{% include %}` |
| planned cards | `{% for r in planned_reports %}` (disabled-with-reason) |
| signal / availability chips | `{{ section.signal|report_signal_chip }}` / `{{ category.availability|report_availability_chip }}` |
| source links | `{% url 'attendance' %}` / `{% url 'sessions' %}` / `{% url 'teacher-performance' %}` / … |

```django
{# _report_card.html — one category card #}
<a class="report-card" data-row data-area="{{ c.id }}" data-availability="{{ c.availability }}" href="{% url c.route_name %}">
  {{ c.availability|report_availability_chip }} <h3>{{ c.title }}</h3> <p>{{ c.desc }}</p>
</a>
```

No whole-page `#app` mount.

## SD9. Enforcement (smoke + structure)

- View-Source / smoke: `reports.html` (+ `.en`) contains baked category cards (each `available` one a real `<a>`) + the overview tiles + the per-area sections + the planned cards; **no `id="app"`**; relative assets; no external request; no raw `⟦key⟧`.
- The G8a grep audit (scope-guard) is clean (no finance/chart/score token).
